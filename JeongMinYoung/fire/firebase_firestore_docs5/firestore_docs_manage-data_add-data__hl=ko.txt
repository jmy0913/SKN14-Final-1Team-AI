Source URL: https://cloud.google.com/firestore/docs/manage-data/add-data?hl=ko

## 서문
이 문서에서는 Firestore에서 개별 문서를 설정, 추가 또는 업데이트하는 방법을 설명합니다. 데이터를 일괄 쓰기하려면 트랜잭션 및 일괄 쓰기를 참고하세요.

## 개요
다음 방법 중 하나로 Firestore에 데이터를 쓸 수 있습니다.

문서 식별자를 명시적으로 지정하여 컬렉션 내의 문서 데이터를 설정합니다.

컬렉션에 새 문서를 추가합니다. 이 경우 Firestore에서 자동으로 문서 식별자를 생성합니다.

자동으로 생성된 식별자로 빈 문서를 만들고 나중에 데이터를 할당합니다.

## 시작하기 전에
데이터를 설정, 추가 또는 업데이트하도록 Firestore를 초기화하려면 다음 단계를 완료해야 합니다.

Firestore 데이터베이스를 만듭니다. 자세한 내용은 Firestore 빠른 시작 중 하나를 참고하세요.

웹 또는 모바일 클라이언트 라이브러리를 사용하는 경우 보안 규칙으로 인증합니다. 자세한 내용은 보안 규칙 시작하기를 참조하세요.

서버 클라이언트 라이브러리 또는 REST API를 사용하는 경우 Identity and Access Management(IAM)로 인증합니다. 자세한 내용은 서버 클라이언트 라이브러리의 보안을 참조하세요.

## 문서 설정
단일 문서를 만들거나 덮어쓰려면 다음 언어별 set() 메서드를 사용합니다.

문서가 없으면 새로 만듭니다. 문서가 있으면 새로 제공한 데이터로 내용을 덮어쓰지만, 다음과 같이 데이터를 기존 문서와 병합하도록 지정한 경우는 예외입니다.

문서가 있는지 확실하지 않은 경우 전체 문서를 실수로 덮어쓰지 않도록 새 데이터를 기존 문서와 병합하는 옵션을 전달하세요. 맵이 포함된 문서의 경우 빈 맵이 있는 필드를 사용해 집합을 지정하면 대상 문서의 맵 필드를 덮어씁니다.

## 커스텀 객체
Map 또는 Dictionary 객체를 사용하여 문서를 표현하기가 불편한 경우가 많으므로 Firestore는 커스텀 클래스를 사용하여 문서를 작성하는 방식도 지원합니다. Firestore에서 객체를 지원되는 데이터 유형으로 변환합니다.

커스텀 클래스를 사용하면 앞의 예시를 다음과 같이 다시 작성할 수 있습니다.

class City {
 final String? name;
 final String? state;
 final String? country;
 final bool? capital;
 final int? population;
 final List<String>? regions;

 City({
 this.name,
 this.state,
 this.country,
 this.capital,
 this.population,
 this.regions,
 });

 factory City.fromFirestore(
 DocumentSnapshot<Map<String, dynamic>> snapshot,
 SnapshotOptions? options,
 ) {
 final data = snapshot.data();
 return City(
 name: data?['name'],
 state: data?['state'],
 country: data?['country'],
 capital: data?['capital'],
 population: data?['population'],
 regions:
 data?['regions'] is Iterable ? List.from(data?['regions']) : null,
 );
 }

 Map<String, dynamic> toFirestore() {
 return {
 if (name != null) "name": name,
 if (state != null) "state": state,
 if (country != null) "country": country,
 if (capital != null) "capital": capital,
 if (population != null) "population": population,
 if (regions != null) "regions": regions,
 };
 }
}
