## 데이터 보안
Firebase 인증과 Firestore 보안 규칙을 사용하여 네이티브 모드의 Firestore에서 데이터를 보호합니다.

다음은 시작하는 데 사용할 수 있는 기본 규칙 세트입니다. Firebase Console의 규칙 탭에서 보안 규칙을 수정할 수 있습니다.

웹, Android 또는 iOS 앱을 프로덕션에 배포하기 전에 앱 클라이언트만 네이티브 모드의 Firestore 데이터에 액세스할 수 있도록 하는 단계를 수행합니다. 앱 체크 문서를 참조하세요.

=== 코드/텍스트 탭 수집 ===
언어: 인증 필요 · 셀#1
// Allow read/write access on all documents to any user signed in to the application
service cloud.firestore {
 match /databases/{database}/documents {
 match /{document=**} {
 allow read, write: if request.auth != null;
 }
 }
}

언어: 잠금 모드 · 셀#1
// Deny read/write access to all users under any conditions
service cloud.firestore {
 match /databases/{database}/documents {
 match /{document=**} {
 allow read, write: if false;
 }
 }
}

언어: 테스트 모드 · 셀#1
// Allow read/write access to all users under any conditions
// Warning: **NEVER** use this rule set in production; it allows
// anyone to overwrite your entire database.
service cloud.firestore {
 match /databases/{database}/documents {
 match /{document=**} {
 allow read, write: if true;
 }
 }
}
