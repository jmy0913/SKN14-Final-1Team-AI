연락처 읽기 및 관리

People API 사용 준비의 단계를 완료하면 주소록을 읽고 관리할 수 있습니다.

다음 코드 샘플은 몇 가지 간단한 요청을 보내는 방법을 보여줍니다. 메서드의 전체 목록은 참조 문서를 참고하세요.

중요: 연락처 기반 사용자만 수정할 수 있습니다. 리소스 이름 "people/me"을 사용하거나 계정 ID를 사용하여 읽은 프로필 기반 사용자는 수정할 수 없습니다.
사용자의 연락처 표시

사용자의 연락처에 있는 사용자 목록을 가져오려면 다음 코드를 사용하세요.

변경된 사용자의 연락처를 나열합니다.
참고: 동기화 요청의 경우 쓰기에 몇 분의 전파 지연이 발생할 수 있습니다. 증분 동기화는 쓰기 후 읽기 사용 사례에 적합하지 않습니다.

동기화 동작에 관한 자세한 내용은 ListConnections를 참고하세요.

사용자의 연락처 검색

사용자의 모든 연락처를 검색하려면 다음 코드를 사용합니다.

중요: 검색은 요청 후에 업데이트되는 지연 캐시를 사용합니다. 클라이언트는 먼저 빈 쿼리가 포함된 준비 검색 요청을 전송하여 캐시에 최신 데이터가 있는지 확인해야 합니다.
참고: 검색은 쿼리의 접두사를 사용자의 필드와 일치시킵니다. 예를 들어 이름이 'foo name'인 사용자는 'f', 'fo', 'foo', 'foo n', 'nam'과 같은 검색어와 일치하지만 'oo n'과는 일치하지 않습니다.

새 연락처를 만들려면 다음 코드를 사용하세요.

요청당 할당량 사용량
중요한 읽기 요청 1회 (연락처 및 프로필 읽기)
